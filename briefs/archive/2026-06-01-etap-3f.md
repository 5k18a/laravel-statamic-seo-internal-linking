# Etap 3f: BubbleMenu dla obrazków w WYSIWYG ✅ ZAMKNIĘTY

Data: 2026-06-01
Status: ✅ Potwierdzone przez użytkownika

## Zakres

Plik: `addons/skalisty/wysiwyg-html-fieldtype/resources/js/components/fieldtypes/WysiwygHtml.vue`

1. Import `BubbleMenu` z `@tiptap/vue-3/menus`
2. Rozszerzenie Image o atrybut `style` przez `.extend({ addAttributes() })`
3. Funkcje `setImageAlign(align)` i `editImageUrl()`
4. `<BubbleMenu>` w template z `shouldShow=({ editor: ed }) => ed.isActive('image')` — przyciski ⬅ ⬛ ➡ ✏ ✕
5. CSS `.wysiwyg-bubble-menu` + `.bubble-btn` w `<style scoped>`

## Kluczowe wnioski

- Import path: `import { BubbleMenu } from '@tiptap/vue-3/menus'` (nie `@tiptap/extension-bubble-menu`)
- BubbleMenu używa `@floating-ui/dom` (nie tippy.js) — prop `options`, nie `tippyOptions`
- `shouldShow` parameter: `({ editor: ed })` — rename aby uniknąć konfliktu z ref scope
- Image extension wymaga `.extend({ addAttributes })` żeby obsługiwał `style` attribute
- `updateAttributes('image', { style: '...' })` działa gdy obraz jest zaznaczony (NodeSelection)

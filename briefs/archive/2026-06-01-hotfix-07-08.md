# HOTFIX 7-8: WYSIWYG dropdown ⋯ z-index + pełna szerokość edytora ✅ ZAMKNIĘTE

Data: 2026-06-01
Status: ✅ Potwierdzone przez użytkownika

## HOTFIX 8 — WYSIWYG pełna szerokość

Klasa `prose` ustawia `max-width: 65ch`. Fix: scoped CSS w `WysiwygHtml.vue`:
```css
.wysiwyg-html-content {
    max-width: none;
}
```

## HOTFIX 7 — dropdown ⋯ w asset-browserze

Teleportowany `[data-reka-popper-content-wrapper]` miał `z-index: auto` (= 0), schowany za modalem (z-index: 10000).

Fix: non-scoped `<style>` w `WysiwygHtml.vue`:
```css
body:has(.wysiwyg-asset-overlay) [data-reka-popper-content-wrapper] {
    z-index: 10001 !important;
}
```

## Kluczowy wniosek

`body:has(.klasa)` jako guard — reguła aktywna tylko gdy modal jest otwarty. Non-scoped CSS wymagane dla elementów teleportowanych poza komponent.

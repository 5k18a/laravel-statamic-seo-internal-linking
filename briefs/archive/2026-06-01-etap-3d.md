# Etap 3d: przycisk 📁 + natywna biblioteka assets ✅ ZAMKNIĘTY

Data: 2026-06-01
Status: ✅ Potwierdzone przez użytkownika (po HOTFIX 4-6)

## Zakres

### PHP: `preload()` w `WysiwygHtml.php`

Metoda `preload()` zwraca `['container' => [...dane kontenera assets...]]`. Samo nadpisanie metody wystarczy — Statamic automatycznie uznaje fieldtype za preloadable przez Reflection.

### Vue: `WysiwygHtml.vue`

1. Prop `meta: { type: Object, default: () => ({}) }`
2. Refy `showAssetBrowser`, `selectedAssets`, `dynamicContainer`
3. Computed `effectiveContainer` (fallback na dynamicContainer)
4. Funkcja `cpUrl(path)` — buduje URL CP z `window.Statamic.$config.get('cpUrl')`
5. Funkcja `onAssetSelected(assets)` — POST do `/cp/assets-fieldtype`, wstawia obraz przez `setImage({ src: url })`
6. Przycisk 📁 otwierający `openBrowser()`
7. `<Teleport to="body">` z `<asset-browser>` + overlay CSS

## Kluczowy wniosek

`asset-browser` jest globalnie zarejestrowany w CP Statamic (`app.component('asset-browser', Browser)`). Komponenty addonów IIFE zarejestrowanych przez `Statamic.component()` mają dostęp do tego samego rejestru Vue — `<asset-browser>` resolwuje się bez importu.

Endpoint `/cp/assets-fieldtype` (POST) zwraca `[{ url, isImage, reference, ... }]` — realny publiczny URL.
